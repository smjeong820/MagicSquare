# /refactor — REFACTOR 구조 정리
**전제 조건: pytest -q 전부 GREEN 상태여야 한다. GREEN이 아니면 즉시 중단하고 보고.**
**추가 입력 없이 실행. 추가 질문 금지.**
## 사전점검 (자동 확인)
1. pytest -q 전체 통과 확인 — FAIL이 하나라도 있으면 중단
2. 정리 대상 파일 목록 확인 (src/ 만)
3. 현재 코드 스멜 후보 식별
## 정리 규칙
- 중복 코드 → 함수 추출 후 재사용
- 매직 넘버(예: 50000, 0.9) → 의미 있는 상수(SSOT)로 추출
- 불명확한 이름 → 의도를 드러내는 이름으로 변경
- 긴 함수 → 단일 책임(SRP) 단위로 분리
- 정리할 게 없으면 "없음"이라 답하고 변경하지 않는다 (과잉 리팩터 금지)
## 절대 금지
- 동작·계약 변경 금지 (기능 추가·수정·삭제)
- 테스트 수정·삭제 금지
- 함수 시그니처 변경 금지
- 구조 변경과 동작 변경을 한 커밋에 섞지 않는다
## 절차
1. 정리 전 pytest -q 실행 → 전부 GREEN 기록
2. 구조 정리 (src/ 만)
3. 정리 후 pytest -q 실행 → 동작 불변 확인
4. 정리할 게 없으면 즉시 "없음" 보고
## 커밋 (내가 요청할 때만)
refactor(계층): [변경 내용] — [계약 ID] [REFACTOR]
예) refactor(entity): THRESHOLD 상수 SSOT 추출 — INV-2 [REFACTOR]
