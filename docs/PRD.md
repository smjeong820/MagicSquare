> **SSOT:** ARRR(Red → Green → Refactor) 전 단계의 단일 진실 원천.  
> 계약표에 없는 기능은 §7 Out of Scope. ID 없는 동작은 구현하지 않는다.

## 2. 진짜 문제 (한 문장)

퍼즐 앱에서 **빈 칸 2개인 4×4 마방진**을 행·열 합(34)으로 빠르게 채우고, 가끔 **틀린 입력을 1~2번 고친 뒤** 다음 문제로 넘어가는 흐름.

---

## 3. R-G-I-O 표

| | 내용 |
|---|---|
| **R (Role · 누가)** | 퍼즐 앱에서 4×4 마방진(0 = 빈 칸, **정확히 2개**)을 푸는 사용자 |
| **G (Goal · 무엇을)** | 두 빈 칸에 들어갈 **올바른 숫자**를 찾아 문제를 끝내고 **다음 문제로 넘어감** |
| **I (Insight · MomTest에서 확인된 사실)** | ① 행·열·대각선 합 34 → **빈 칸 있는 행**부터 시작 (M-3) ② 대부분 **직관·~1분** (M-2, M-4) ③ 행·열 중 하나는 0이 **1개뿐** → 막히지 않음 (M-5) ④ 맞추면 **별도 검증 없이** 다음 문제 (M-6) ⑤ **잘못 입력** → 틀림 → **수정 1~2회** (M-8~M-11) |
| **O (Outcome · 성공 기준)** | §4 AC-1~AC-5 (MomTest 증거 1:1) |

---

## 4. 성공 기준 (AC) — MomTest 증거와 1:1 연결

| AC ID | 성공 기준 | MomTest 증거 (인용) |
|---|---|---|
| **AC-1** | 유효 퍼즐 입력 시 **빈 두 칸 정답** `(행, 열, 값)` 2쌍을 산출한다. | M-2 「**1분 정도?**」 |
| **AC-2** | **마법상수 = 34**; 0이 **1개**인 행 또는 열에서 `34 − (나머지 칸 합)`으로 **첫 빈 칸**을 확정한다. | M-3 「**빈 칸이 있는 행 기준으로 시작**」 |
| **AC-3** | 빈 칸 **정확히 2개**, **1~16 중복 없음**, 모든 **행·열·대각선 합 = 34**를 만족하는 **유일한** 숫자 쌍을 반환한다. | M-5 「**0이 두개… 행, 열 둘 중 하나는… 0만… 안풀리는 문제는 없었어**」 + 명시 규칙 |
| **AC-4** | 잘못된 후보(합≠34, 중복, 대각선 불일치)는 **거부** 가능; 최종 두 값은 **전체 INV 검증 통과**. | M-8~M-11 「**틀렸다고 해**」→「**수정해서 다시 입력**」→「**1~2번?**」 |
| **AC-5** | Boundary 출력은 **빈 두 칸 숫자만** 제공한다 (추가 확인·설명 없음). | M-6 「**다음 문제로 넘어갔어**」 |

## 5. 계약표

| ID | 계약 | 근거레벨 | 계층 | 테스트 아이디어 |
|---|---|---|---|---|
| **INV-1** | 4×4 마방진 마법상수 = **34** | L1 | Entity | `magic_constant(4) == 34` |
| **INV-2** | 완성 격자 **모든 행** 합 = 34 | L1 | Entity | 4행 각 `sum(row) == 34` |
| **INV-3** | 완성 격자 **모든 열** 합 = 34 | L1 | Entity | 4열 각 `sum(col) == 34` |
| **INV-4** | **주·부대각선** 합 = 34 | L1 | Entity | `sum(main_diag) == sum(anti_diag) == 34` |
| **INV-5** | 채워진 칸 값 **1~16**, **중복 없음** | L1 | Entity | `all(1<=v<=16)`, `len(set)==len` |
| **INV-6** | 입력 격자 **0(빈 칸) = 정확히 2개** | L0/L1 | Entity | `count_zeros(grid) == 2` |
| **INV-7** | 행 또는 열 중 **적어도 하나**는 0이 **1개**뿐 *(퍼즐 제약)* | L2 | Entity | `any(row_zeros==1) or any(col_zeros==1)` |
| **E-1** | 4×4 아님 / None / 비정형 타입 → **거부** | L0 | Entity + Boundary | 3×3, `None` → 예외 |
| **E-2** | 0 개수 ≠ 2 → **거부** | L0 | Entity + Boundary | 0·1·3개 → 에러 |
| **E-3** | 0·1~16 밖 값(음수 포함) → **거부** | L0 | Entity + Boundary | `-1`, `17` → 에러 |
| **E-4** | 0 제외 **중복 값** → **거부** | L0/L1 | Entity | `[8,8,...]` → 에러 |
| **E-5** | 빈 칸 **후보**가 INV-2~5 위반 → **틀림(거부)** | L2 | Entity + Boundary | 행합≠34 후보 → `False` |
| **AC-1** | 유효 퍼즐 → 빈 두 칸 `(r,c,v)` **2쌍** 반환 | L2/L3 | Entity | conftest 격자 → 기대 좌표·값 |
| **AC-2** | INV-7 만족 → **유일해** | L3 | Entity | golden + 전 INV 재검증 |
| **AC-3** | 0이 1개인 **행 우선** 첫 빈 칸 확정 | L2 | Entity | 행·열 모두 단일 0 시 행 우선 |
| **AC-4** | 잘못된 후보 **거부** + 최종 정답 **전체 검증** | L2/L3 | Entity + Boundary | E-5 + solve 후 INV-* |
| **AC-5** | Boundary 응답 = **빈 두 칸 숫자만** | L3 | Boundary | `"7, 12"` 형식; 해설 없음 |

## 7. Out of Scope — 만들지 않을 기능과 MomTest 근거

계약표(§5)에 ID가 없는 동작은 모두 Out of Scope.

| OOS ID | 만들지 않을 동작 | MomTest / PRD 근거 |
|---|---|---|
| **OOS-1** | 힌트·단계별 풀이 UI | M-4 「**직관적으로 해결**」— 힌트 니즈 없음 |
| **OOS-2** | 정답 후 **별도 검증·확인** 화면 | M-6 「**다음 문제로 넘어갔어**」 |
| **OOS-3** | 풀이 **소요 시간** 측정·제한 | M-2 「1분」은 사용자 행동; 시스템 AC 아님 |
| **OOS-4** | 퍼즐 **생성·난이도** 조절 | M-1 맥락: 기존 문제 **풀기**만 |
| **OOS-5** | 빈 칸 **3개 이상** / **5×5** 등 n×n | 명시 규칙: 4×4, 빈 칸 **2개** |
| **OOS-6** | 앱 실행 계기·푸시·랭킹 | M-7 「**그건 중요하지 않아**」 |
